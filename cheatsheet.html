📋 Plan & Manage (20-25%)

Responsible AI — 6 Principes
  • Fairness — geen bias/discriminatie
  • Reliability & Safety — consistent + veilig
  • Privacy & Security — databescherming
  • Inclusiveness — toegankelijk voor iedereen
  • Transparency — uitlegbaar
  • Accountability — mensen verantwoordelijk
🧠 FRPITA
AI Services: Multi vs Single Resource
  • Multi-service: 1 key, meerdere services → minder beheer
  • Single-service: apart per service → eigen RBAC/budget
  • Custom subdomain verplicht voor Entra ID/token auth
  • Soft-delete: purge nodig voor hergebruik naam
Content Safety — HVSS
Cat.Levels
Hate, Violence, Sexual, Self-harmsafe → low → medium → high
  • Default: medium+ geblokkeerd
  • Threshold Low = strengst (alleen safe door)
  • Threshold High = meest permissief (alleen high geblokt)
  • Blokkering = HTTP 400, niet beleefde weigering
  • Blocklists: exacte woorden, altijd geblokt
  • Prompt Shields: jailbreak + indirect injection
  • Groundedness: hallucination check (≠ content filter!)
Foundry (voorheen AI Studio)
  • Hub = gedeelde container (connections, compute)
  • Project = werkruimte in Hub (deployments, flows, agents)
  • Model Catalog deployment:
    • Azure OpenAI — per token, MS-managed
    • Serverless/MaaS — per token, Llama/Mistral
    • Managed Compute — per uur, dedicated VM
  • Limited Access: gezichts-ID features → MS goedkeuring
Networking & Auth
  • Managed Identity: geen credentials, automatische rotatie
  • Private Endpoints: AI services in VNET isolatie
  • API key auth uitgeschakeld → Bearer token via az account get-access-token
  • RBAC rollen: Cognitive Services User (consume), Contributor (manage)

🧠 Quota = per model per regio (niet per deployment!)
Prompt Engineering
  • Zero-shot: directe vraag, geen voorbeelden
  • Few-shot: voorbeelden in prompt
  • CoT: "denk stap voor stap" → betere redenering
  • Rollen: system → user → assistant
  • Temperature: 0 = deterministisch, 1+ = creatief
  • Bij RAG: temperature weinig effect
Function Calling — 3 Stappen
  1. Define — tools in request
  2. Decide — model kiest → finish_reason: "tool_calls"
  3. Execute — JIJ voert uit, stuurt result als tool message
  • Model voert NOOIT zelf uit!
  • Parallel calls: elke call eigen tool_call_id
  • tool_choice: auto | none | required | specifiek
Assistants API vs Chat Completions
Chat CompletionsAssistants
StateStateless (jij)Stateful (service)
ToolsFunction calling+ Code Interpreter + File Search
ExecSynchroonAsync (poll/stream)
  • Flow: Assistant → Thread → Message → Run → Poll
  • requires_action → submit_tool_outputs
  • Threads: geen token limiet (auto truncate)
On Your Data (RAG built-in)
  • Koppel AI Search index aan OpenAI deployment → automatisch RAG met citations
  • Auth: API key (dev) of Managed Identity (prod)
  • RBAC: OpenAI → Search = Search Index Data Reader · Search → OpenAI = Cognitive Services OpenAI User
  • Search API access = "Both" of "RBAC" (niet API-only!)

🔍 Knowledge Mining (15-20%)

AI Search Pipeline
  • Data Source → Indexer → (Skillset) → Index
  • Push (jij stuurt data) vs Pull (indexer op schema)
  • Skillset: OCR → Merge → Split → Embedding
  • Merge skill: essentieel! Combineert tekst + OCR output
Search Types
TypeMechanismeBest voor
BM25Inverted indexExacte termen, codes
VectorHNSW, cosine simSemantisch, synoniemen
HybridBM25 + Vector → RRFBijna altijd best
🧠 Hybrid = parallel (niet sequentieel!), RRF combineert
Semantic Ranker
  • GEEN search type → het is L2 re-ranking (post-processing)
  • Herordent top-N resultaten op semantische relevantie
  • Voegt captions + answers toe
  • Werkt alleen op tekstvelden, niet op vectoren
  • Standard tier+, apart geprijsd
Chunk-level Index
  • Index Projections: chunks als aparte records
  • Key field: searchable=true + analyzer=keyword
  • sourceContext moet matchen met split skill output
  • projectionMode: skipIndexingParentDocuments
  • Security trimming: filterveld + OData filter bij query

🤖 Agentic (5-10%)

Agent vs Chatbot
  • Agent: autonoom, plant stappen, meerdere rondes
  • Agent loop: Observe → Think → Act → herhaal
  • Agent Service tools: Code Interpreter, File Search, Azure Functions
  • Grounding: AI Search, Bing, SharePoint, Fabric
