🔮 Generative AI

Azure OpenAI Service 15-20% exam
Cognitive Service · Managed API
Toegang tot OpenAI-modellen via Azure met enterprise security, content filtering en RBAC.
Modellen: GPT-4o, GPT-4.1, GPT-4.1-mini, GPT-3.5-turbo, o1, o3-mini (reasoning), DALL-E 3 (images), Whisper (audio→tekst), text-embedding-ada-002 / text-embedding-3-* (embeddings)
Gebruik: Chat/completion, RAG generatie, function calling, embeddings genereren, image generatie, audio transcriptie
Assistants API
Azure OpenAI feature
Persistent threads met state management. Ondersteunt tools (code interpreter, file search, function calling) en automatische context management.
Onderdelen: Assistant → Thread → Message → Run → Tool outputs
Gebruik: Stateful conversaties, code uitvoeren, file analysis, multi-turn met tools
On Your Data
Azure OpenAI feature
Koppel Azure AI Search direct aan een chat completion endpoint. Automatische RAG zonder eigen orchestratie.
Gebruik: Snelle RAG setup, chatbot op eigen documenten, citaties uit zoekresultaten
Microsoft Foundry (voorheen AI Studio)
Platform · Hub/Project model
Unified platform voor AI development. Hub = gedeelde resources, Project = werkruimte. Model Catalog met 1600+ modellen.
Deployment types: MaaS (pay-per-token, serverless), Managed Compute (dedicated VM), Global/Standard/Provisioned
Gebruik: Model selectie & deployment, Prompt Flow, evaluaties, agent orchestratie
Foundry Agent Service 5-10% exam
Foundry feature
Bouw AI agents met tools (Bing Search, Azure AI Search, file search, Azure Functions, OpenAPI). Agent loop: model beslist autonoom welke tools nodig zijn.
Gebruik: Autonome agents, multi-step taken, real-time data ophalen, multi-agent orchestratie

👁️ Computer Vision

Azure AI Vision (Image Analysis 4.0) 10-15% exam
Cognitive Service · REST API
Analyse van afbeeldingen: captions, tags, objecten, mensen, OCR (Read), smart crops, dense captions. Gebaseerd op Florence foundation model.
Features: caption, denseCaptions, tags, objects, people, smartCrops, read
Gebruik: Beeldbeschrijving, object detectie, tekst uit afbeeldingen (OCR), thumbnail generatie
Custom Vision
Standalone service · Training + Prediction endpoint
Train eigen classificatie- of detectiemodellen met je eigen afbeeldingen. Ondersteunt compact domains voor edge/ONNX export.
Types: Multiclass classification (1 tag/image), Multilabel classification (meerdere tags), Object Detection (bounding boxes)
Gebruik: Productinspectie, custom herkenning, edge deployment (compact models → ONNX/TensorFlow/CoreML)
Face API
Cognitive Service · REST API
Gezichtsdetectie (attributen, landmarks) is vrij beschikbaar. Identificatie, verificatie en groepering vereisen Limited Access goedkeuring.
Operaties: Detect (vrij), Identify, Verify, Group, Find Similar Limited Access
Gebruik: Gezichtsherkenning, identity verification, aanwezigheidscontrole
Video Indexer
Standalone service · ARM-connected accounts
Automatische extractie van inzichten uit video: transcript, sprekers, topics, gezichten, merken, emoties, scenes, keyframes.
Gebruik: Video doorzoekbaar maken, ondertiteling, content moderatie, media analyse
Spatial Analysis
Edge container · Azure IoT / Docker
Real-time analyse van beweging in videostreams via edge containers. Geen video opslag in de cloud.
Operaties: PersonCount, PersonCrossingLine, PersonCrossingPolygon, PersonDistance
Gebruik: Winkelbezetting, social distancing, verkeersstromen, zones bewaken

💬 Language (NLP)

Azure AI Language 15-20% exam
Cognitive Service · REST API
Tekst analyse: sentiment, key phrases, NER, PII detectie + redactie, entity linking (Wikipedia disambiguatie), taaldetectie.
Pre-built features: Sentiment Analysis, Key Phrase Extraction, NER, PII Detection, Entity Linking, Language Detection, Opinion Mining
Gebruik: Klanttevredenheid, PII masking, entiteiten extraheren, meertalige content analyseren
CLU (Conversational Language Understanding)
Language feature · Custom model
Train custom modellen om intents en entities te herkennen in gesprekstekst. Opvolger van LUIS.
Onderdelen: Intents, Entities (learned, list, prebuilt, regex), Utterances (min. 15 per intent aanbevolen)
Gebruik: Chatbot intent herkenning, commando interpretatie, slot filling
Custom Question Answering
Language feature · Knowledge Base
FAQ knowledge base met multi-turn conversaties, chit-chat, en orchestration workflow. Opvolger van QnA Maker.
Bronnen: URLs, documenten, handmatige QA-paren, multi-language (per KB of per QA-paar)
Gebruik: FAQ bots, helpdeskautomatisering, gestructureerde Q&A
Custom Text Classification
Language feature · Custom model
Train modellen om tekst te classificeren in je eigen categorieën.
Types: Single-label (1 categorie per tekst), Multi-label (meerdere categorieën)
Gebruik: Document routing, ticket classificatie, content tagging
Custom NER (Named Entity Recognition)
Language feature · Custom model
Extraheer domein-specifieke entiteiten uit tekst met je eigen gelabelde data.
Gebruik: Contract analyse, medische dossiers, domein-specifieke extractie
Translator
Cognitive Service · REST API
Tekst- en documentvertaling, transliteratie, en custom vertaalmodellen.
Types: Text Translation (sync), Document Translation (async/batch), Custom Translator (min. 10K parallel zinnen, BLEU score 0-100)
Gebruik: Meertalige apps, document vertaling, domein-specifieke vertaling

🎙️ Speech

Speech Services
Cognitive Service · SDK + REST
Spraak naar tekst, tekst naar spraak, spraakvertaling, en sprekerherkenning.
STT varianten: Real-time, Batch (async, SAS URL), Custom Speech (eigen data/model)
TTS: Neural voices, SSML (pitch, rate, emphasis, breaks, phoneme)
Overig: Speech Translation, Speaker Recognition (verification + identification), Custom Neural Voice Limited Access
Gebruik: Transcriptie, voorlezen, realtime vertaling, biometrie
Keyword Recognition
On-device · Speech SDK
Wake word detectie die volledig lokaal draait op het device. Geen cloud calls nodig.
Gebruik: "Hey Cortana"-achtige activatie, privacy-sensitive scenario's
Intent Recognition
Speech SDK + CLU
Combineert STT en CLU in één aanroep: spraak → tekst → intent + entities.
Gebruik: Voice commands, spraakgestuurde bots

🔍 Search & Knowledge Mining

Azure AI Search 15-20% exam
PaaS · Managed search platform
Full-text, vector en hybrid zoeken met AI-verrijking via skillsets. Indexing pipeline: Data Source → Skillset → Indexer → Index.
Zoektypen:
• BM25 (full-text, keyword matching)
• Vector (cosine similarity, HNSW algoritme)
• Hybrid (BM25 + vector parallel, RRF fusie)
• Hybrid + Semantic Ranker (+ LLM re-ranking, captions, answers)
Gebruik: RAG retrieval, document search, e-commerce, kennisbank
Built-in Skills (Skillset Pipeline)
AI Search feature
AI-verrijking tijdens indexering. Skills voegen metadata toe aan documenten.
Belangrijke skills:
• OcrSkill — tekst uit afbeeldingen/gescande PDFs
• SplitSkill — chunking (tekst splitsen)
• AzureOpenAIEmbeddingSkill — vector embeddings genereren
• MergeSkill — velden samenvoegen
• EntityRecognitionSkill — NER
• EntityLinkingSkill — Wikipedia disambiguatie
• DocumentExtractionSkill — file content extractie
• ShaperSkill — output structureren voor Knowledge Store
Gebruik: OCR pipeline, chunk-level indexing, verrijkte metadata toevoegen
Knowledge Store
AI Search feature · 3 projectie-types
Sla verrijkte data op buiten de index voor downstream analyse.
Projecties:
• Table → Azure Table Storage (gestructureerd, Power BI)
• Object → Blob Storage als JSON
• File → Blob Storage als bestanden (afbeeldingen, genormaliseerde docs)
Gebruik: Analytics, Power BI dashboards, data export, downstream processing

📄 Document Intelligence

Azure AI Document Intelligence 15-20% exam
Cognitive Service · REST API
Extractie van tekst, structuur, tabellen en velden uit documenten. Voorheen Form Recognizer.
3 categorieën:
• General: Read (OCR), Layout (structuur + tabellen), General Document
• Prebuilt: Invoice, Receipt, W-2, ID Document, Health Insurance Card, Business Card
• Custom: Template (vaste layout), Neural (variabele layout), Composed (routing via Classifier)
Gebruik: Facturenverwerking, ID verificatie, formulier extractie, custom document types
Content Understanding
Azure AI Service · Multi-modal Preview
LLM-backed, multi-modale analyse van documenten, afbeeldingen, video én audio. Schema-based extractie.
Gebruik: Complexe documenten met mixed media, video/audio analyse, gestructureerde output

🛡️ Content Safety & Responsible AI

Azure AI Content Safety
Cognitive Service · REST API
Detectie van schadelijke content in tekst en afbeeldingen.
HVSS categorieën: Hate, Violence, Sexual, Self-harm (severity 0-6, threshold 0/2/4/6)
Features: Blocklists (custom woorden), Prompt Shields (jailbreak + indirect injection), Groundedness Detection (hallucination check), Protected Material Detection
Gebruik: Content filtering in Azure OpenAI, UGC moderatie, prompt beveiliging

🔧 Platform & Orchestratie

Prompt Flow / Workflows
Foundry feature
Visuele orchestratie van LLM-pipelines. Bouwen, testen, evalueren en deployen als endpoint.
Flow types: Standard (LLM chains), Chat (conversation), Evaluation (metrics berekenen)
Evaluatie metrics: Groundedness, Relevance, Coherence, Fluency, Similarity, F1
Gebruik: RAG pipelines bouwen, A/B testing met variants, kwaliteitsmonitoring

🧠 Deployment & Auth Patterns

Multi-service vs Single-service Resource
Multi-service: Eén resource, één key voor Vision + Language + Speech etc. Eén endpoint.
Single-service: Dedicated resource per service. Eigen key, eigen endpoint. Meer controle over billing en access.
Authenticatie
API Key: Ocp-Apim-Subscription-Key header. Simpel maar minder veilig.
Entra ID (Bearer token): Managed Identity of service principal. Enterprise-grade. Vereist custom subdomain.
RBAC rollen: Cognitive Services User (consume), Cognitive Services Contributor (manage), specifieke rollen per service.
Containers (Offline)
Sommige services draaien als Docker container voor on-premise/edge.
Connected: periodieke billing call naar Azure.
Disconnected: volledig offline (commitment plan, Limited Access).
Exam tip: "Docker" of "container" = offline scenario. "Mismatch" error = verkeerde API key.
Networking
Public: open internet (default).
Selected Networks: specifieke VNets/IPs (firewall rules).
Disabled + Private Endpoint: alleen via VNet, geen publiek verkeer. Meest restrictief.
Exam tip: Private Endpoint + Disabled ≠ Selected Networks!
