1. ML Fundamentals
Supervised vs Unsupervised Learning
  • Supervised: gelabelde data → model leert input→output mapping (classificatie, regressie)
  • Unsupervised: geen labels → model vindt patronen zelf (clustering, dimensionaliteitsreductie)
  • Reinforcement: agent leert door trial & error met beloningen
LLM Basics
  • Transformer architectuur: attention mechanisme, parallelle verwerking van tokens
  • Tokens: kleinste eenheid (~4 chars EN, ~2-3 chars NL), bepalen kosten en context limiet
  • Context window: maximale input+output tokens (GPT-4o: 128K input)
  • Temperature: 0 = deterministisch, 1+ = creatief/random. Bij gegronde RAG: weinig effect omdat retrieval het antwoord stuurt
2. Prompt Engineering
Rollen
  • system — instructies, persona, regels (altijd eerste message)
  • user — de vraag/input van de gebruiker
  • assistant — het antwoord van het model (of few-shot voorbeelden)
Technieken
  • Zero-shot: directe vraag zonder voorbeelden
  • Few-shot: voorbeelden van gewenste input→output in de prompt
  • Chain-of-Thought (CoT): "Denk stap voor stap" — model toont redenering, betere resultaten bij complexe vragen
  • Query translation: NL query → EN vertaling in system prompt voor EN documenten
  • Grounding: model beperken tot gegeven context. Te streng = weigert alles, te los = hallucineert
💡 Grounding balans: itereer op de prompt. Start streng, versoepel tot het model correct antwoordt zonder te hallucineren.
3. Responsible AI
Microsoft's 6 Principes
  • Fairness — geen discriminatie of bias
  • Reliability & Safety — consistent en veilig gedrag
  • Privacy & Security — databescherming
  • Inclusiveness — toegankelijk voor iedereen
  • Transparency — uitlegbaar, gebruiker weet dat het AI is
  • Accountability — mensen blijven verantwoordelijk

6. Function Calling
3-Stappen Flow
1. Define  — Jij definieert tools (functions) in de request
2. Decide  — Model beslist of het een tool wil aanroepen → finish_reason: "tool_calls"
3. Execute — Jij voert de functie uit en stuurt resultaat terug als tool message

Belangrijk: het model voert NOOIT zelf functies uit — het vraagt jouw applicatie om dat te doen.

Tool Definitie
{
  "tools": [{
    "type": "function",
    "function": {
      "name": "get_weather",
      "description": "Haal het huidige weer op voor een locatie",
      "parameters": {
        "type": "object",
        "properties": {
          "location": { "type": "string", "description": "De stad" }
        },
        "required": ["location"]
      }
    }
  }]
}
Tool Result Terugsturen

Na een tool_calls response stuur je de hele conversatie terug inclusief:

  • Het assistant bericht met de originele tool_calls
  • Een tool message per call, gekoppeld via tool_call_id
{ "role": "tool", "tool_call_id": "call_abc123", "content": "{\"temperature\": 12}" }
Parallel Tool Calls
  • Model kan meerdere tool calls in één response doen (bijv. weer in 2 steden)
  • Elke call heeft een eigen id — stuur voor elke call een apart tool message terug
  • Werkt ook met verschillende tools (bijv. get_weather + get_news tegelijk)
tool_choice Parameter
WaardeGedrag
"auto"Model beslist zelf (default)
"none"Mag geen tools gebruiken — valt terug op eigen kennis
"required"Moet minstens één tool aanroepen
{"type":"function","function":{"name":"..."}}Forceer een specifieke tool

Kernconcepten

Flow
1. Create Assistant  (eenmalig — model + instructions + tools)
2. Create Thread     (per gesprek)
3. Add Message       (user message toevoegen)
4. Create Run        (assistant verwerkt de thread)
5. Poll Run Status   (async! queued → in_progress → ...)
6. ├── completed        → haal assistant message op
   └── requires_action → submit tool outputs → poll weer

Ingebouwde Tools
ToolWat het doet
Code InterpreterDraait Python in een sandbox — kan berekeningen doen, files lezen/schrijven, grafieken maken

Belangrijke Details
  • Threads hebben geen token limiet — de API truncate automatisch
  • Runs zijn async — poll de status of gebruik streaming
  • Code Interpreter output kan files bevatten (grafieken als PNG, CSV, etc.)
  • Assistants worden opgeslagen per resource — kosten niks tot je een Run start
  • File Search gebruikt een Vector Store — apart object, koppelbaar aan assistant of thread

Evaluation Metrics (LLM-as-Judge)
MetricWat meet hetDiagnose bij lage score
GroundednessIs antwoord gebaseerd op context?Model hallucineert → betere system prompt
RelevanceBeantwoordt het de vraag?Verkeerde chunks → retrieval probleem
CoherenceIs het logisch/leesbaar?Slechte prompt template
FluencyGrammaticaal correct?Model/taal mismatch
SimilarityLijkt op ground truth?F1-achtige vergelijking

Diagnose-regel: Groundedness hoog + Relevance laag = retrieval probleem (verkeerde chunks). Groundedness laag = hallucination (model probleem).

Batch Runs & Evaluation Runs
  • Batch run: voer flow uit op dataset (JSONL) → genereer antwoorden
  • Evaluation run: voer eval flow uit tegen batch resultaten → scores per rij
  • Column mapping: koppel eval inputs aan batch outputs via --column-mapping
  • CLI: pf run create --flow ... --data ... --run <baseline> --name ...
Connections
  • Bewaar credentials los van flow code
  • Types: AzureOpenAI, OpenAI, CognitiveSearch, Custom
  • auth_mode: meid_token → Entra ID (geen API key nodig)
  • CLI: pf connection create --file connection.yaml
Deployment
  • Flow deployen als managed online endpoint in Foundry
  • Versioning, traffic splitting, autoscaling
  • Monitoring via Application Insights
SDK vs Portal
  • New Foundry UI: Prompt Flow vervangen door "Workflows" (agent orchestratie)
  • Classic Foundry: Prompt Flow UI beschikbaar (hub-based projects only)
  • SDK/CLI: promptflow Python package + pf CLI — werkt altijd
  • VS Code extension: visuele DAG editor

API Response bij Blokkering
HTTP 400 — content_filter error
├── content_filter_result
│   ├── hate: {filtered: false, severity: "safe"}
│   ├── sexual: {filtered: true, severity: "high"}  ← triggerde
│   ├── violence: {filtered: false, severity: "safe"}
│   └── self_harm: {filtered: false, severity: "safe"}
├── custom_blocklists: {filtered: true, details: [{id: "...", filtered: true}]}
└── jailbreak: {detected: false}

Let op: een echte blokkering geeft een 400 error, niet een beleefde weigering van het model.

Blocklists
  • Custom lijsten van exacte woorden/zinnen
  • Altijd geblokkeerd, ongeacht severity score
  • Use case: concurrentnamen, domein-specifieke termen
Prompt Shields
  • Detecteert jailbreak pogingen (user omzeilt system prompt)
  • Detecteert indirect prompt injection (schadelijke instructies in documenten/data)
Groundedness Detection
  • Checkt of output gebaseerd is op meegegeven context
  • ≠ Content filtering (schadelijkheid) — dit is hallucination check
Content Safety Resource
  • Azure AI Content Safety = apart resource (Microsoft.CognitiveServices/accounts, kind: ContentSafety)
  • Content filters in Azure OpenAI zijn ingebouwd — geen apart resource nodig
  • Standalone Content Safety resource: voor custom tekst/beeld analyse buiten OpenAI

12. Azure AI Search
Indexing Pipeline
Data Source → Indexer → (Skillset) → Index

Data Source: Blob Storage, SQL, Cosmos DB, etc.
Indexer:     Motor die data leest, change tracking, schema-based of on-demand
Skillset:   Verrijkingspipeline (keten van skills)
Index:      Schema met velden (type + attributen: searchable, filterable, sortable, facetable, retrievable)
Skillset — Enrichment Tree
Document
├── content (originele tekst)
├── normalized_images[]
│   └── [OCR skill] → text per afbeelding
└── [Merge skill] → merged_content (tekst + OCR samen!)
    └── [Split skill] → chunks[]
        └── [Embedding skill] → vector per chunk

Merge skill is essentieel: zonder merge verlies je óf de originele tekst óf de OCR-output bij gescande documenten.

Search Types
TypeMechanismeSterk bijZwak bij
Full-Text (BM25) Inverted index, TF-IDF scoring Exacte termen, codes, namen Synoniemen, parafrases, cross-language
Vector (KNN) HNSW algoritme, cosine similarity op embeddings Semantisch begrip, synoniemen, cross-language Exacte matches, namen, codes
Hybrid BM25 + Vector parallel → RRF combinatie Bijna altijd beste keuze —
Query Pipeline (Hybrid + Semantic Ranker)
Query
├── Tokenize → BM25 search (inverted index) → ranked list A
├── Embed   → Vector search (HNSW)          → ranked list B
└── RRF: merge list A + B → top-N resultaten
    └── Semantic Ranker: re-rank top-N → final results + captions + answers
Semantic Ranker
  • Geen search type — het is een L2 re-ranking stap (post-processing)
  • Herordent top resultaten met een deep learning model op semantische relevantie
  • Voegt captions (meest relevante passage) en answers (direct antwoord) toe
  • Werkt alleen op tekstvelden, niet op vectoren
  • Beschikbaar vanaf Standard tier, apart geprijsd per 1000 queries
Security Trimming
  • Document-level access control via een filterveld (bijv. allowed_groups)
  • Bij indexeren: vul per document welke groepen toegang hebben
  • Bij query: OData filter meesturen → $filter=allowed_groups/any(g: g eq 'engineering')
  • App haalt groepen op via Microsoft Entra ID en bouwt het filter
  • Niet: aparte indexes per afdeling (schaalt niet)
Index Vullen: Push vs Pull
  • Push: jij stuurt data naar de index via de REST API — meer controle, real-time
  • Pull: indexer draait op schema (bijv. elke 5 min) en checkt data source op wijzigingen
Chunk-level Index met Index Projections

Standaard maakt een indexer één record per document. Voor RAG wil je één record per chunk — zodat vector search specifieke fragmenten teruggeeft, niet hele documenten.

Document-level vs Chunk-level

AspectDocument-levelChunk-level
Records in index1 per document1 per chunk
Embedding1 vector per heel doc1 vector per fragment
Vector search retourneertHele documentSpecifiek relevant fragment
Context window gebruikSnel volPrecies genoeg

Index Projections

De indexProjections in de skillset zorgen dat elke chunk als apart record in een aparte index terechtkomt:

"indexProjections": {
  "selectors": [{
    "targetIndexName": "idx-chunks",
    "parentKeyFieldName": "parent_id",
    "sourceContext": "/document/chunks/*",
    "mappings": [
      { "name": "chunk",  "source": "/document/chunks/*" },
      { "name": "vector", "source": "/document/chunks/*/vector" },
      { "name": "title",  "source": "/document/metadata_storage_name" }
    ]
  }],
  "parameters": {
    "projectionMode": "skipIndexingParentDocuments"
  }
}

Index Schema voor Chunks

VeldTypeKeySearchableAnalyzerNoot
chunk_idEdm.String✅✅keywordAuto-generated, keyword analyzer verplicht!
parent_idEdm.StringLink naar brondocument, filterable
chunkEdm.String✅De chunk tekst
vectorCollection(Edm.Single)✅1536d (ada-002), HNSW profiel
titleEdm.String✅Bestandsnaam van parent
💡 Key field van een chunk index moet searchable=true + analyzer=keyword hebben — anders weigert Azure de index projection.
💡 sourceContext moet exact matchen met de output van je split skill (bijv. /document/chunks/*, niet /document/pages/*).
💡 projectionMode: skipIndexingParentDocuments = alleen chunks in de target index, parent docs overslaan.
Semantic Configuration
  • Definieer op de index welke velden de semantic ranker moet gebruiken
  • Content field: het tekstveld dat gerankt wordt (bijv. chunk)
  • Title field: optioneel, weegt mee in relevantie (bijv. title)
  • Bij query: queryType=semantic + semanticConfiguration=sc-default
Chunk Overlap
  • Split skill: pageOverlapLength (bijv. 200 chars op 2000 chars chunks = 10% overlap)
  • Voorkomt context verlies op chunk-grenzen — zinnen die net op de split vallen zitten in beide chunks
  • Minimale storage/embedding overhead, grote verbetering in retrieval kwaliteit
13. RAG Pattern
Retrieval-Augmented Generation
User Query
  → Retrieval (AI Search: hybrid + semantic ranker)
  → Top-K chunks als context
  → Generation (Azure OpenAI: system prompt + context + query)
  → Gegrond antwoord
  • Chunking is essentieel: te groot = irrelevante context, te klein = context verlies
  • Temperature heeft weinig effect bij goed gegronde RAG — retrieval stuurt het antwoord
"On Your Data" (Azure OpenAI)
  • Ingebouwde RAG feature: koppel een AI Search index aan een Azure OpenAI deployment
  • Doet automatisch: query → search → inject resultaten als context → LLM antwoord met citations
  • Configureerbaar: search type (keyword/vector/hybrid), semantic ranker aan/uit

16. Geleerde Lessen
💡 OCR is nodig voor gescande PDF's — standaard extractie haalt geen tekst uit afbeeldingen
💡 Tier limieten: Basic = max 16MB per blob, S1 = max 128MB
💡 RBAC debugging: controleer altijd of de managed identity de juiste rollen heeft op alle resources
💡 Prompt iteratie: begin streng (geen hallucinaties), versoepel geleidelijk
💡 Quota is gedeeld per model per regio, niet per deployment
💡 Standard deployments auto-upgraden naar nieuwere model versies
💡 Content filtering (HVSS) ≠ grounding — twee aparte concepten
💡 Groundedness check = hallucination detection (is het antwoord gebaseerd op de context?)
💡 Chunk-level index key field: moet searchable=true + analyzer=keyword — anders weigert Azure de index projection
💡 Index projection sourceContext moet exact matchen met de split skill output (check de targetName!)
💡 AI Search API access control: "API key only" blokkeert managed identity auth — zet op "Both" of "RBAC"
💡 Managed identity Object IDs kunnen verschillen tussen portals (classic vs Foundry) — altijd de juiste ID verifiëren
💡 Function calling: het model voert NOOIT zelf functies uit — het genereert alleen de aanroep, jouw app voert uit
💡 tool_choice: "none" = model weet dat tools bestaan maar mag ze niet gebruiken — valt terug op eigen kennis (of bluf)
💡 Parallel tool calls: elke call heeft een eigen tool_call_id — stuur voor ELKE call een apart tool message terug
💡 Assistants API: threads hebben geen token limiet — de service truncate automatisch, jij hoeft niks te doen
💡 Chat Completions = stateless (jij beheert history), Assistants API = stateful (service beheert threads)
💡 Org policy kan API keys blokkeren — gebruik dan Bearer token via az account get-access-token + Entra RBAC
💡 Microsoft Foundry = hernoemd van Azure AI Foundry = hernoemd van Azure AI Studio — zelfde product, zelfde URL
💡 Foundry maakt stiekem resources aan (Storage, Key Vault) wanneer je een Hub/Project maakt
💡 New Foundry portal toont alleen Foundry projects — voor andere resource types heb je classic nodig
💡 Content Safety filters ≠ Groundedness detection: filters blokkeren schadelijke content, groundedness checkt hallucinaties
💡 Content filters in Azure OpenAI zijn ingebouwd — geen apart Content Safety resource nodig
💡 Blocklists blokkeren altijd, ongeacht severity score — voor exacte woorden/zinnen
💡 Agent Service bouwt voort op Assistants API maar voegt Bing, AI Search, Functions en OpenAPI als extra tools toe
💡 Fine-tuning is geen agent capability — agents gebruiken tools, geen model training
💡 Custom subdomain is verplicht voor token/Entra ID authenticatie bij Cognitive Services — zonder krijg je "provide custom subdomain" error
💡 Image Analysis 4.0 Florence features (Caption, Dense Captions, Smart Crops) zijn niet in alle regio's beschikbaar — check regionale beschikbaarheid
💡 Soft-deleted Cognitive Services resources moeten eerst gepurged worden voordat je dezelfde naam kunt hergebruiken
💡 Caption = beschrijvende zin, Tags = losse keywords — niet verwarren
💡 Custom Vision Compact domain = voor edge/offline deployment, exporteerbaar naar ONNX/CoreML/TensorFlow
💡 Precision = "ben ik precies?" (hoge precision, weinig false positives), Recall = "mis ik er geen?" (hoge recall, weinig false negatives)
💡 Spatial Analysis PersonCount = hoeveel nu aanwezig, PersonCrossingLine = hoeveel passeerden — verschil tussen aanwezigheid vs passage
💡 Face API: detectie is vrij, alles met identiteit (verification, identification, grouping) vereist Limited Access goedkeuring
💡 CLU zit in de Language resource — geen aparte resource nodig
💡 NER categoriseert entiteiten, Entity Linking disambigueert en linkt naar Wikipedia
💡 PII Detection is de enige Text Analytics feature die tekst kan redacten
💡 Document Translation is async/batch via Blob Storage — niet synchroon zoals Text Translation
💡 Custom Neural Voice vereist Limited Access goedkeuring
💡 Orchestration workflow: CLU project dat intents routeert naar CLU of Custom QA projecten
💡 Keyword Recognition draait on-device (wake word) — Intent Recognition is cloud-based (STT + CLU in één stap)
💡 Opinion Mining is geen aparte feature maar een parameter op Sentiment Analysis (opinionMining: true)
💡 Custom Translator vereist minimum 10.000 parallel sentences; kwaliteit gemeten met BLEU score
💡 Connected containers (Text Analytics, Speech, Translator, CLU) bellen periodiek Azure voor billing — alleen Custom Vision compact is echt offline
💡 Custom Text Classification: single-label (1 cat/doc) vs multi-label (meerdere) — verschil met CLU: classificatie = hele document, CLU = intents uit utterances
17. Begrippenlijst
Agent
AI systeem dat autonoom beslissingen neemt, tools gebruikt en multi-step taken uitvoert — meer dan een reactieve chatbot
Agent Service
Microsoft Foundry managed service voor agents — bouwt voort op Assistants API met extra tools (Bing, Search, Functions, OpenAPI)
Assistant (API)
Configuratie-object in de Assistants API — bevat model, instructions en tools. Herbruikbaar over meerdere threads.
Blocklist
Custom lijst van exacte woorden/zinnen die altijd geblokkeerd worden in Content Safety, ongeacht severity score
BM25
Best Match 25 — standaard ranking algoritme voor full-text search (term frequency × inverse document frequency)
Chunk Overlap
Overlappende tekst tussen opeenvolgende chunks (bijv. 200 chars) — voorkomt context verlies op splitgrenzen
Caption
Image Analysis feature die één beschrijvende zin genereert voor het hele beeld — gebruikt Florence model, ideaal voor alt-text
BLEU Score
Bilingual Evaluation Understudy — meet vertaalkwaliteit (0-100) door vergelijking met menselijke referentievertalingen
CLU
Conversational Language Understanding — opvolger van LUIS, begrijpt intents en entities uit vrije tekst binnen de Language resource
Custom Text Classification
Language service feature om documenten te classificeren (single-label of multi-label) — verschil met CLU: hele document vs individuele utterances
Chunking
Tekst opdelen in kleinere stukken zodat elke chunk apart geëmbedded en geretrievet kan worden
CoT
Chain-of-Thought — prompt techniek waarbij het model stapsgewijs redeneert voor betere antwoorden
Code Interpreter
Ingebouwde Assistants API tool die Python draait in een sandbox — kan berekeningen, files en grafieken maken
Content Filtering
Azure OpenAI feature dat schadelijke content blokkeert in 4 categorieën (HVSS)
Cosine Similarity
Maat voor gelijkenis tussen twee vectoren (1 = identiek, 0 = geen relatie)
Custom Vision
Azure service voor het trainen van eigen image classification of object detection modellen — compact domains voor edge export
Dense Captions
Image Analysis feature die meerdere beschrijvende zinnen genereert, elk voor een andere regio in het beeld, met bounding boxes
Embedding
Numerieke vectorrepresentatie van tekst die semantische betekenis vastlegt
Enrichment Tree
Boomstructuur van skill inputs/outputs binnen een AI Search skillset
Entity Linking
Text Analytics feature die entiteiten disambigueert en koppelt aan Wikipedia — "Mars" → planeet vs merk
Face API
Azure AI service voor gezichtsdetectie, verificatie en identificatie — detectie is vrij, identificatie/verificatie vereist Limited Access goedkeuring
Few-shot
Prompt techniek met voorbeelden van gewenste input→output in de prompt
File Search
Ingebouwde Assistants API tool die geüploade documenten doorzoekt via een Vector Store (automatisch chunking + embeddings)
Florence
Microsoft's multimodal foundation model achter Image Analysis 4.0 — maakt Caption, Dense Captions en Smart Crops mogelijk
Function Calling
Mechanisme waarbij het model tools/functies kan aanroepen — het model beslist en vraagt, jouw app voert uit
Grounding
Model beperken tot alleen antwoorden op basis van gegeven context (voorkomt hallucinaties)
Groundedness
Mate waarin een antwoord gebaseerd is op de aangeleverde context (≈ hallucination check)
Hallucination
Model genereert informatie die niet in de context staat — klinkt overtuigend maar is verzonnen
HNSW
Hierarchical Navigable Small World — algoritme voor snelle approximate nearest neighbor search in vector spaces
Hub
Microsoft Foundry container voor gedeelde resources (connections, compute, policies) — bevat Projects als werkruimtes
HVSS
Hate, Violence, Sexual, Self-harm — de 4 content filtering categorieën in Azure OpenAI
Hybrid Search
Full-text (BM25) + vector search parallel, gecombineerd via RRF
Indexer
AI Search component dat data uit een source haalt, door skillset duwt, en in de index stopt
Index Projection
Skillset feature dat chunks als aparte records in een (aparte) index projecteert — essentieel voor chunk-level RAG
Inverted Index
Datastructuur die woorden mapt naar documenten (zoals de index achteraan in een boek)
KNN
K-Nearest Neighbors — zoek de K dichtstbijzijnde vectoren bij een query vector
L2 Re-ranker
Tweede-fase ranking model dat initiële zoekresultaten herordent op semantische relevantie
Keyword Recognition
Speech feature die on-device luistert naar een wake word ("Hey Cortana") — geen cloud latency
Limited Access
Microsoft goedkeuringsproces voor gevoelige AI capabilities (gezichtsherkenning, identificatie) — aanvragen via formulier
MaaS
Models as a Service — serverless deployment in Foundry Model Catalog, pay-per-token, geen infra beheer (voor Llama, Mistral, etc.)
Managed Identity
Azure-beheerde identiteit voor resources — geen credentials nodig, automatische rotatie
Merge Skill
AI Search skill die originele tekst en OCR-output samenvoegt tot één tekstveld
NER
Named Entity Recognition — detecteert en categoriseert entiteiten in tekst (personen, locaties, datums, organisaties)
OCR
Optical Character Recognition — tekst herkennen uit afbeeldingen/gescande documenten
On Your Data
Azure OpenAI feature dat automatisch RAG doet: koppelt een Search index aan een GPT deployment voor gegronde antwoorden met citations
Project
Microsoft Foundry werkruimte binnen een Hub — bevat deployments, prompt flows, evaluations en agents
PII Detection
Text Analytics feature die persoonsgegevens detecteert en kan redacten — enige feature die tekst vervangt door ***
Opinion Mining
Parameter op Sentiment Analysis (opinionMining: true) die per-aspect sentiment geeft — "hotel=positive, ontbijt=negative"
Precision
Van alle positieve voorspellingen, welk percentage klopt? Hoge precision = weinig false positives
Prompt Shields
Content Safety feature dat jailbreak-pogingen en indirect prompt injection detecteert
PTU
Provisioned Throughput Units — gereserveerde capaciteit voor Azure OpenAI (voorspelbare kosten)
RAG
Retrieval-Augmented Generation — relevante documenten ophalen en meegeven als context aan een LLM
Recall
Van alle werkelijke positieven, welk percentage wordt gevonden? Hoge recall = weinig false negatives
Run
Assistants API: uitvoering van een assistant op een thread — asynchroon, met statussen (queued → in_progress → completed/requires_action)
RRF
Reciprocal Rank Fusion — methode om ranked lists uit verschillende search types te combineren
Semantic Ranker
AI Search post-processing stap die zoekresultaten herordent met een deep learning model + captions/answers toevoegt
SSML
Speech Synthesis Markup Language — XML-opmaak om TTS-output te sturen (pauzes, snelheid, toon, nadruk, stem kiezen)
Spatial Analysis
Azure AI Vision feature voor real-time people counting en movement detection via camera feeds — draait als Docker container op edge met NVIDIA GPU
Skillset
AI Search verrijkingspipeline — keten van skills die documenten transformeren tijdens indexering
Security Trimming
Document-level access control via filtervelden op query-time (niet aparte indexes)
Temperature
Parameter die randomness/creativiteit van LLM output regelt (0 = deterministisch, 1+ = creatief)
TF-IDF
Term Frequency × Inverse Document Frequency — hoe belangrijk is een woord voor een document
Thread
Assistants API: persistente conversatie die messages bevat — service beheert de history, geen token limiet
Token
Kleinste teksteenheid voor LLM's (~4 karakters EN), bepaalt kosten en context limiet
TPM
Tokens Per Minute — quota-eenheid voor Azure OpenAI, gedeeld per model per regio
tool_call_id
Unieke ID die een tool result koppelt aan de oorspronkelijke tool call — essentieel bij parallel calls
tool_choice
Parameter die bepaalt of/hoe het model tools mag gebruiken: auto (default), none, required, of specifieke functie
Transformer
Neural network architectuur achter moderne LLM's — attention mechanisme voor parallelle verwerking
Vector Search
Zoeken op basis van semantische afstand tussen embedding vectoren (cosine similarity)
Video Indexer
Azure service die video's analyseert en doorzoekbaar maakt — extraheert transcripts, speakers, topics, faces, brands met tijdstempels
Zero-shot
Prompt zonder voorbeelden — direct een vraag stellen aan het model
